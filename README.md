# java-racingcar-concurrency

이 프로젝트는 기존 `java-racingcar-8` 싱글스레드 기반 자동차 경주 게임을
**멀티스레드 + 동시성 안전 구조**로 리팩터링한 구현입니다.

김영한님의 *실전 자바 - 고급 1편(멀티스레드와 동시성)* 강의를 완강한 뒤,
학습한 내용을 실제 미션에 적용해보고자 마련한 프로젝트로,
자동차 이동 로직을 병렬 처리하고, 공유 자원의 동시성 문제를 해결하는 것을 목표로 합니다.

주요 목표는 다음과 같습니다.
- ExecutorService 기반의 **스레드 풀 적용**
- 자동차 이동의 **병렬 처리**
- 상태 공유를 위한 **동시성 제어(Atomic, Lock, synchronized)**
- 안전한 레이스 결과 수집을 위한 **스레드 간 협력 구조 설계**
- 싱글스레드 코드 구조를 **동시성 친화적으로 리팩터링**

기존 미션의 요구사항을 유지하면서, 성능과 구조 측면에서 더 확장성 있고
실무적인 형태의 레이싱 엔진으로 발전시키는 것을 목표로 합니다.

---
## 자동차 경주 게임

초간단 자동차 경주 게임을 구현한 프로젝트입니다.

- 사용자로부터 경주할 자동차 이름들을 입력 받습니다. (이름은 쉼표(,) 기준으로 구분 / 5자 이하만 가능)
- 사용자로부터 이동할 횟수를 입력 받습니다.
- 한 이동 당 각각의 자동차는 0에서 9 사이의 무작위 값을 받아, 무작위 값이 4 이상일 경우 전진합니다.
- 차수별 실행 결과를 출력합니다.
- 게임을 완료한 후 누가 우승했는지 출력합니다. 우승자는 한 명 이상일 수 있습니다.
- 사용자가 잘못된 값을 입력할 경우 애플리케이션은 종료됩니다.

## 기능 목록

1. 경주할 자동차 이름 입력 받기
2. 시도할 횟수 입력 받기
3. 잘못된 값을 입력할 경우 IllegalArgumentException 발생시킨 후 애플리케이션 종료
   - 이름이 5자 초과인 경우 (한글/영어 모두)
   - 자동차 1대만 입력한 경우 (경주 게임이므로 2대 이상의 자동차가 참여해야한다.)
   - 빈 토큰이 들어간 경우(예: ",pobi,jun" , "pobi,,jun" , "pobi,jun,")
   - 공백 입력
   - 시도할 횟수가 숫자가 아닌 경우
   - 시도할 횟수가 0 이하인 경우 (최소 1회 이상이어야 한다.)
4. 입력에 따라 각각의 자동차 구현하기
5. 게임 구현하기
   - 0~9 사이의 무작위 값 구하기
   - 무작위 값이 4이상일 경우 자동차 전진
6. 차수별 실행 결과 출력하기
7. 최종 우승자 출력하기
   - 최종 우승자가 2명 이상인 경우 쉼표(,)로 구분해 출력하기
